# Blog 文章迁移 Skill

## 用途
将其他位置的 Markdown 文件迁移到博客项目中，自动添加合适的 frontmatter 并放置到正确的目录。

## 使用方法
当用户要求迁移文章时，请按以下步骤执行：

### Step 1: 获取源文件路径
向用户确认源文件的完整路径。

### Step 2: 读取源文件
使用 Read 工具读取源文件内容。

### Step 3: 分析文章内容
分析文章内容，确定：
- 文章标题（从 # 一级标题或内容推断）
- 文章描述（从内容摘要或开头段落提取）
- 相关标签（从内容关键词推断）
- 合适的目标目录

### Step 4: 确定目标位置
博客目录结构如下，根据文章内容选择合适位置：

```
docs/专题系列/
├── AI 打铁铺/              # AI 相关深度技术文章
├── Web 前端/
│   ├── 重学前端/           # 前端基础知识系统学习
│   ├── 疑难杂症/           # 前端问题解决方案
│   └── JS 速查表/          # JS 速查表
└── 其他/
    ├── 博客配置/           # 博客配置相关
    └── 我的开源/           # 开源项目介绍
```

**目录选择原则：**
- AI、LLM、Prompt、Agent、AI Coding 相关 → `AI 打铁铺/`
- 前端技术学习 → `Web 前端/重学前端/`
- 前端 Bug 修复、兼容性问题 → `Web 前端/疑难杂症/`
- JS 速查 → `Web 前端/JS 速查表/`
- 其他内容询问用户

### Step 5: 生成 Frontmatter
根据目标目录生成对应的 frontmatter：

```yaml
---
group:
  title: [目录名称]
tags:
  - [标签1]
  - [标签2]
sidebar: false
sticky: [优先级，1最高]
date: [当前日期 YYYY-MM-DD HH:mm:ss]
recommend: [推荐级别]
title: [文章标题]
description: [文章描述，100字以内]
cover: ""
---
```

**Frontmatter 字段说明：**
- `group.title`: 与目标目录名称一致
- `tags`: 3-5个相关标签
- `sidebar`: 通常为 false（非目录索引页）
- `sticky`: 置顶优先级，1 最高，新文章建议 1-3
- `date`: 当前时间
- `recommend`: 推荐级别，与 sticky 对应
- `title`: 文章标题，可以加前缀如 "专题名 | 标题"
- `description`: 简短描述，用于 SEO 和预览
- `cover`: 封面图 URL，可为空

### Step 6: 生成文件名
文件名规则：
- 使用英文或拼音
- 多个单词用 `-` 连接
- 简洁明了，体现文章主题
- 示例：`figma-to-code-last-mile.md`

### Step 7: 确认迁移
向用户展示：
- 源文件路径
- 目标文件路径
- 生成的 frontmatter
- 文件名

获得用户确认后执行迁移。

### Step 8: 执行迁移
1. 使用 Write 工具创建新文件
2. 写入 frontmatter + 原始内容
3. 确认文件创建成功

### Step 9: 提示用户
提醒用户：
- 如需立即看到效果可能需要重启开发服务器
- 检查文章在博客中的显示效果
- 如有需要可以手动调整 frontmatter

## 示例对话

用户：帮我把 `/path/to/article.md` 迁移到博客

AI：
1. 读取源文件内容
2. 分析得出：标题为 "xxx"，内容关于 AI Coding
3. 建议放置到 `AI 打铁铺/` 目录
4. 生成 frontmatter 并展示
5. 确认后执行迁移
6. 完成并提示用户

## 注意事项
- 不修改原始文章内容，仅添加 frontmatter
- 如果源文件已有 frontmatter，需要合并或替换
- 优先询问用户对目录选择和文件名的意见
- 迁移完成后发送飞书通知
